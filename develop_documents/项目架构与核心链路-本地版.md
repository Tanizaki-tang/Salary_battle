# 劳资拉扯模拟器：项目架构与核心链路（本地版 / Electron 离线包）

## 1. 版本定位

- 交付形态：`Electron` 离线安装包（Windows 优先）。
- 目标：无外网依赖也可完整演示，现场稳定性优先。
- 运行模式：
  - `api`：Electron 内置 FastAPI（本地回环地址）
  - `mock`：纯前端固定流程（零 API 依赖）

---

## 2. 架构总览

## 2.1 技术栈

- 桌面容器：`Electron`
- 前端：`Vue 3 + Vite + TypeScript`
- 后端：`FastAPI + Uvicorn`
- 数据库：`SQLite`（本地文件）
- 语音识别：`Sherpa-ONNX`（本地模型）

## 2.2 进程结构

- **Electron Main 进程**：
  - 启动窗口
  - 拉起 FastAPI 子进程
  - 关闭应用时回收子进程
- **Renderer 进程（Vue）**：
  - 展示 UI
  - 调用 Runtime Adapter
- **Python 子进程（FastAPI）**：
  - 处理游戏接口、落库、语音识别

---

## 3. 目录对应关系（本地版重点）

- Electron：`desktop/main`
  - `main.ts`
  - `backend_process.ts`
  - `preload.ts`
- 前端：`frontend/src`
  - `runtime/api_runtime_adapter.ts`
  - `runtime/mock_runtime_adapter.ts`
  - `runtime/battle_runtime_adapter.ts`
- 后端：`backend/app`
- 资源目录：`resources/sherpa-onnx-models`、`resources/seed`

---

## 4. 核心函数与输入输出

## 4.1 Electron 侧

- `startBackend() -> ChildProcess`
  - 输入：无
  - 输出：后端子进程句柄
- `stopBackend() -> void`
  - 输入：无
  - 输出：无（回收进程）

## 4.2 后端侧（同在线版）

- `run_text_turn(session_state, text_payload) -> TurnResult`
- `run_voice_turn(session_state, audio_payload) -> VoiceTurnResult`
- `advance_game_flow(session_state, turn_result) -> SessionState`
- `settle_session(session_state) -> SettleResult`
- `save_session_result(user_id, session_result, session_id) -> PersistResult`

---

## 5. 核心链路（本地）

## 5.1 应用启动链路（api 模式）

1. 用户双击启动 Electron。
2. `main.ts` 调用 `startBackend()` 拉起 FastAPI（如 `127.0.0.1:18080`）。
3. Renderer 加载前端页面。
4. 前端调用 `GET /api/v1/health` 校验后端健康。
5. 进入对局流程。

## 5.2 对局链路（api 模式）

1. `POST /api/v1/sessions` 创建本地对局。
2. 文本：`text-turn`；语音：`voice-turn`。
3. 回合结束后 `settle`。
4. 本地 `SQLite` 写入 `user_id + final_score`。

## 5.3 对局链路（mock 模式）

1. 前端 runtime 切换到 `mock_runtime_adapter`。
2. 读取 `frontend/src/mock/*.json`。
3. 不发任何 HTTP 请求，直接按固定树推进。
4. 生成本地结果页，完成演示闭环。

---

## 6. 本地版故障兜底策略

- 后端启动失败：自动切 `mock` 模式，确保演示不停。
- 语音识别异常：回退文字输入。
- 模型加载慢：预热提示 + 首次加载缓存。
- 端口冲突：提供可配置端口（默认 `18080`）。

---

## 7. 本地版最小验收

- Electron 启动后能打开主界面。
- `api` 模式可跑完整一局并落库。
- `mock` 模式在无后端进程时仍可完整跑通。
- 退出应用后 FastAPI 子进程被正确回收。
