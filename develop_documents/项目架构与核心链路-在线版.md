# 劳资拉扯模拟器：项目架构与核心链路（在线版）

## 1. 版本定位

- 交付形态：在线 `HTML` 网站（浏览器访问 URL）。
- 目标：保证多用户可访问、前后端分离部署、接口链路清晰。
- 运行模式：优先 `api` 模式（真实后端）；可保留前端 `mock` 兜底。
- 说明：离线 `Electron` 版本延期到下一阶段，本期不纳入交付范围。

---

## 2. 架构总览

## 2.1 技术栈

- 前端：`Vue 3 + Vite + TypeScript`
- 后端：`FastAPI + Uvicorn`
- 数据库：`SQLite`（黑客松阶段）
- 语音识别：`Sherpa-ONNX`（服务端调用）

## 2.2 逻辑分层

- **前端展示层**：页面与交互（开始页、对话页、结果页）。
- **前端运行时层**：`BattleRuntimeAdapter` 统一调用入口。
- **后端 API 层**：路由与协议适配（`/api/v1/*`）。
- **后端编排层**：`GameFlowOrchestrator` 统一调度模块。
- **功能模块层**：
  - `flow_controller`（状态推进、结算、落库）
  - `text_battle`（文字回合）
  - `voice_battle`（语音回合）
- **数据层**：`SQLite` 持久化 `user_id + final_score`。

## 3. 目录对应关系（在线版重点）

- 前端主目录：`frontend/src`

- 三模块：
  - `backend/app/modules/flow_controller`
  - `backend/app/modules/text_battle`
  - `backend/app/modules/voice_battle`

## 4. 核心函数与输入输出

- `run_text_turn(session_state, text_payload) -> TurnResult`
  - 输入：当前状态 + 文本策略/文本内容
  - 输出：HR 回复、指标变化、是否结束
- `run_voice_turn(session_state, audio_payload) -> VoiceTurnResult`
  - 输入：当前状态 + 语音输入信息
  - 输出：ASR 文本 + 同构 `TurnResult`
- `advance_game_flow(session_state, turn_result) -> SessionState`
  - 输入：旧状态 + 回合结果
  - 输出：新状态
- `settle_session(session_state) -> SettleResult`
  - 输入：结算状态
  - 输出：最终报价、分数、评级、复盘建议
- `save_session_result(user_id, session_result, session_id) -> PersistResult`
  - 输入：用户 ID + 结算结果 + 对局 ID
  - 输出：是否落库成功

## 5. 核心链路（在线）

## 5.1 启动链路

1. 启动前端静态站点。
2. 启动 FastAPI 服务。
3. 前端通过 `VITE_API_BASE_URL` 指向后端。
4. 前端先调用 `GET /api/v1/health` 做可用性探测。

## 5.2 对局主链路（文字）

1. `POST /api/v1/sessions` 创建对局。
2. `POST /api/v1/sessions/{id}/text-turn` 推进回合。
3. 重复回合直到结束条件满足。
4. `POST /api/v1/sessions/{id}/settle` 结算并落库。

## 5.3 对局主链路（语音）

1. 前端上传音频至 `POST /api/v1/sessions/{id}/voice-turn`。
2. 后端 `voice_battle` 调用 `Sherpa-ONNX` 得到转写文本。
3. 复用统一回合规则得到 `TurnResult`。
4. 状态机更新并返回前端。

---

## 6. 在线版协作边界（1 人核心链路 + 2 人核心功能）

- 成员 A（核心链路）：维护 `API 路由 + orchestrator + flow_controller + 落库链路`。
- 成员 B（核心功能 1）：只维护 `text_battle`（策略解析与文字回合规则）。
- 成员 C（核心功能 2）：只维护 `voice_battle`（ASR 接入与语音回合规则）。
- 约束：模块间通过契约和共享类型交互，避免跨模块直接改逻辑。

---

## 7. 在线版最小验收

- `health` 可用。
- `sessions` / `text-turn` / `voice-turn` / `settle` 全链路可通。
- `settle` 后数据库可查询到 `user_id + final_score`。
- 前端可在 `api` 模式完整跑完一局。
